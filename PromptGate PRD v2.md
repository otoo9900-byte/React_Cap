# **PromptGate PRD (Product Requirements Document)**

## **Intelligent Prompt Gateway (v2.0)**

## **1\. 개요**

**PromptGate**는 사용자의 단순하고 모호한 요구사항을 AI와의 '스무고개'식 대화 인터페이스를 통해 전문가 수준의 프롬프트로 자동 변환해 주는 지능형 미들웨어 서비스입니다. **Dify**의 시각적 워크플로우와 **Python(Flask)** 기반의 견고한 백엔드, 그리고 React 프론트엔드를 결합하여, 누구나 챗봇과 대화하는 것만으로 최상급의 LLM(Gemini, GPT 등) 결과물을 얻을 수 있도록 돕습니다.

### **핵심 가치 제안**

* **직관적인 대화형 인터뷰 (Interactive Interview):** 사용자는 프롬프트 작성법을 전혀 몰라도 됩니다. 챗봇이 먼저 의도를 파악하고, 필요한 정보(예: 예산, 기간, 테마 등)를 친절하게 역질문하여 데이터를 수집합니다.  
* **프롬프트 자동 조립 (Prompt Assembly):** 대화를 통해 얻은 파편화된 답변들을 종합하여, 백그라운드에서 Chain-of-Thought 등 고급 프롬프트 엔지니어링 기법이 적용된 '완벽한 프롬프트'로 자동 변환합니다.  
* **유연한 워크플로우 (Dify Integration):** 하드코딩 없이 Dify를 활용해 질문 분기(IF/ELSE)와 프롬프트 템플릿을 시각적으로 관리하여 확장성을 극대화합니다.  
* **비용 및 보안 최적화 (Nice-to-have):** 사전 대화는 가벼운 모델로 처리하고 완성된 프롬프트만 고성능 모델에 전송해 토큰 비용을 아끼며, 필요시 민감 정보(PII)를 필터링합니다.

## **2\. 타겟 사용자**

### **주 사용자**

* **AI 활용이 서툰 일반인 및 학생:** ChatGPT 화면을 띄워놓고 어떻게 지시해야 원하는 대답(기획서, 여행 일정 등)이 나올지 막막함을 느끼는 사용자.  
* **업무 효율을 높이고 싶은 직장인:** 빠르고 정확한 결과물이 필요하지만, 매번 복잡한 프롬프트를 작성하기 귀찮은 사용자.

### **페인포인트**

* "대충 물어보면 대충 대답해 줘서, 결국 내가 원하는 결과가 나올 때까지 계속 질문을 고쳐 써야 한다."  
* "프롬프트를 잘 쓰는 공식(페르소나, 출력 형식 지정 등)이 있다는데, 배우기도 복잡하고 매번 적용하기 번거롭다."

## **4\. 기술 스택 (Tech Stack)**

### **AI 및 워크플로우 엔진 (Core)**

* **Dify:** 노드 기반의 LLM 워크플로우 설계. 의도 분석, 조건부 역질문 분기, 프롬프트 템플릿 매핑 및 메인 LLM 호출 담당.

### **백엔드 프레임워크**

* **Python (Flask):** React 프론트엔드와 Dify API 사이를 중계하는 경량 API 서버. 사용자 세션 관리 및 DB 연동 담당.  
* **PostgreSQL:** 프롬프트 생성 히스토리 및 사용자 데이터 저장.

### **프론트엔드 프레임워크**

* **React (Vite):** 컴포넌트 기반 UI 개발. (채팅창과 결과창이 동시에 보이는 Split View 구조)  
* **Tailwind CSS:** 직관적이고 깔끔한 반응형 채팅 인터페이스 구현.

### **5.2. 모듈 간 데이터 흐름 (Data Flow)**

1. **프론트엔드 (ChatInterface.jsx):** 사용자가 "부산 여행 계획 짜줘" 메시지 입력.  
2. **백엔드 (api/chat_routes.py):** 메시지 수신 -> security_service.py를 거쳐 민감 정보 마스킹 -> dify_client.py를 통해 Dify API 서버로 요청 전송.  
3. **Dify 워크플로우:** 의도 파악 노드 -> 조건 분기 노드 -> 역질문 생성(정보 부족 시) 또는 프롬프트 조립(정보 충분 시) 수행.  
4. **스트리밍 수신 (services/dify_client.py):** Dify로부터 반환되는 청크(Chunk) 데이터를 SSE로 수신하여 즉시 프론트엔드로 중계 전달.  
5. **상태 업데이트 (usePromptGateStore.js):** 상태 업데이트를 통해 좌측 채팅창에 스무고개 질문을 띄우거나, 우측 결과창에 최종 생성된 결과를 실시간 타이핑 효과로 렌더링.

## **6\. 핵심 서비스 시나리오 및 라우팅 전략**

1. **사용자 요청:** "부산 여행을 갈려고 하는데 여행 계획을 짜줘."  
2. **Phase 1 (정보 분석):** Dify의 첫 번째 노드가 입력을 분석. \[주제: 여행\]은 파악했으나 필수 정보(예산, 기간, 테마)가 누락되었음을 인지.  
3. **Phase 2 (역질문 루프):**  
   * 챗봇: "부산 여행이군요\! 예산은 어느 정도로 생각하시나요?"  
   * 사용자: "50만 원."  
   * 챗봇: "몇 박 몇 일 일정이고, 어떤 테마를 좋아하시나요?"  
   * 사용자: "2박 3일, 바다 휴양 위주로."  
4. **Phase 3 (프롬프트 조립 및 최종 생성):**  
   * Dify 엔진이 수집된 정보 {목적지: 부산, 예산: 50만 원, 기간: 2박 3일, 테마: 바다 휴양}를 결합.  
   * "당신은 전문 여행 플래너입니다. 총예산 50만 원으로 바다 휴양 위주의 2박 3일 부산 일정을 표로 작성해 주세요..." 라는 완성형 프롬프트로 변환.  
   * 이를 Gemini/GPT에 전달하여 완벽한 여행 계획표를 반환.

## **7\. 사용자 스토리 (User Stories)**

### **US-001: 대화형 정보 수집**

* **As a** AI 활용 초보자는  
* **I want to** 내가 원하는 바를 대충 말해도 챗봇이 "예산은요?", "기간은요?" 하고 필요한 부분만 딱딱 짚어서 물어봐주기를 원한다.  
* **So that** 복잡하게 머리 쓸 필요 없이 편안하게 대화만으로 완벽한 여행 계획표나 기획서를 받을 수 있다.

### **US-002: 완성된 프롬프트 확인 및 재사용**

* **As a** 업무 효율을 중시하는 사용자는  
* **I want to** 챗봇이 내 대답을 바탕으로 도대체 '어떤 멋진 프롬프트'를 만들어냈는지 화면에서 직접 확인하고 복사할 수 있기를 원한다.  
* **So that** 나중에 비슷한 작업을 할 때 이 프롬프트를 템플릿처럼 바로 재사용할 수 있다.

## **8\. 향후 로드맵 (Roadmap)**

* **1\~4주차:** Flask 기반 기본 API 서버 세팅, React 채팅 UI 프로토타입 개발. Dify 워크스페이스 구축 및 여행 기획 Chatflow 노드 초안 설계.  
* **5\~8주차:** Dify 내 IF/ELSE 조건 분기를 활용한 '스무고개(Slot-filling)' 로직 구현. React 프론트엔드와 Flask 백엔드를 거쳐 Dify API 완벽 연동.  
* **9\~12주차:** 프롬프트 템플릿 조립기 고도화 및 최종 LLM(Gemini 등) 결과 출력 스트리밍 구현. 프롬프트 저장(아카이브) 기능 DB 연동.  
* **13\~16주차:** 다양한 도메인(이메일 작성, 마케팅 등)으로 챗봇 워크플로우 시나리오 확장. 시스템 통합 테스트 및 캡스톤 최종 발표 준비.
