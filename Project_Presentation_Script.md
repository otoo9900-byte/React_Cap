# 🚀 PromptGate 프로젝트 시스템 구조 및 핵심 로직 발표 대본

**발표자용 참고사항:** 이 문서는 프로젝트의 '시스템 아키텍처'와 '핵심 기능의 기술적 구현 로직'을 깊이 있게 설명하는 데 초점을 맞추어 재작성된 스크립트입니다. "어떻게 동작하는가?", "어떤 로직을 썼는가?"를 명확히 전달할 수 있도록 구성되었습니다.

## 1. 프로젝트 개요 (Introduction)

**[발표 스크립트]**
"안녕하세요. **PromptGate (지능형 프롬프트 게이트웨이)** 프로젝트 발표를 맡은 OOO입니다. 
PromptGate는 사용자의 단순하고 모호한 요구사항을 입력받아, 부족한 부분을 AI가 스무고개처럼 '역질문'하여 파악하고 최종적으로 전문가 수준의 프롬프트로 자동 조립해 주는 서비스입니다. 

오늘은 이 서비스가 내부적으로 **어떤 아키텍처**로 구성되어 있고, 이 스무고개 대화와 프롬프트 조립이 **어떤 로직**으로 구현되었는지 시스템 구조를 중점적으로 설명해 드리겠습니다."

## 2. 시스템 아키텍처 요약 (System Architecture)

**[발표 스크립트]**
"저희 시스템은 확장성과 유지보수성을 고려하여 철저하게 역할이 분리된 3계층(3-Tier) 구조를 띠고 있습니다."

```mermaid
graph LR
    subgraph "1. Frontend (React/Zustand)"
        UI_Chat[Chat Interface\n(대화 렌더링)]
        UI_Result[Result View\n(정규식 파싱 및 조립)]
    end

    subgraph "2. Backend Middleware (Flask)"
        API[Flask API Router\n/api/chat]
        Logic[보안 / 노이즈 필터링]
    end

    subgraph "3. AI Core (Dify Engine)"
        Flow[Chatflow 노드 로직\n(의도 파악 & 역질문)]
        LLM[(LLM)]
    end

    UI_Chat <-->|메시지 송수신| API
    API <-->|API Key 인증 & 데이터 전달| Flow
    Flow <-->|조건 분기 & 프롬프트 생성| LLM
```

*   **프론트엔드 (사용자 경험 계층)**: React와 Zustand를 사용하여 챗봇 대화창과 최종 프롬프트 결과창을 분리한 Split-View 구조를 구현했습니다.
*   **백엔드 (보안 및 중계 계층)**: Python Flask를 활용하여 클라이언트에서 API Key가 탈취되는 것을 막는 미들웨어 역할을 합니다.
*   **AI 코어 (추론 및 제어 계층)**: 단순한 1회성 API 호출이 아닌, Dify의 노드(Node) 기반 Chatflow 엔진을 활용하여 대화의 흐름을 능동적으로 제어합니다.

## 3. 핵심 로직 구현 상세 (How It Works)

**[발표 스크립트]**
"그럼 지금부터 이 3가지 계층이 실제로 상호작용하며 어떻게 '지능형 역질문'과 '프롬프트 파싱'을 이끌어내는지 세부 로직을 설명하겠습니다."

### 1) AI 코어: Dify Chatflow를 활용한 '스무고개' 조건 분기 로직
"가장 핵심이 되는 AI의 스무고개 역질문 대화는 Dify의 챗플로우(Chatflow) 로직으로 구현했습니다.

*   **동작 방식**: 사용자가 '부산 여행 계획 짜줘'라고 입력하면, 시스템은 단순 프롬프트로 LLM에 넘기지 않습니다. 먼저 **'의도 분석 노드(Intent Recognition Node)'**를 거쳐 사용자가 여행 계획을 원한다는 것을 파악합니다.
*   **IF/ELSE 조건 분기 로직(Slot-filling)**: 그 다음, 완벽한 프롬프트를 만들기 위해 필요한 필수 변수(예산, 여행 기간, 선호 테마 등)에 공백(Null)이 있는지 검사합니다.
*   만약 변수 중 하나라도 비어있다면, 로직은 LLM 결과 창출 노드가 아닌 **'역질문 생성 노드'**로 빠지게 설계했습니다. 즉, "예산 정보가 없습니다. 예산은 얼마로 생각하시나요?"라고 되묻게 됩니다.
*   사용자가 모든 조건을 대답하여 변수가 채워지면, 비로소 **'프롬프트 템플릿 조립 노드'**로 진입하여 "예산 N만원, 기간 M일..." 형식의 완벽한 프롬프트를 배경에서 자동 조립해 냅니다."

### 2) 백엔드: 안전한 데이터 중계 및 노이즈 필터링 로직
"프론트엔드와 AI 코어를 이어주는 Flask 백엔드는 보안과 최적화를 담당합니다.

*   **동작 방식**: 프론트엔드 컴포넌트(`chatClient.js`)가 백엔드의 `/api/chat` 라우트로 요청을 보냅니다.
*   **인증 로직 캡슐화**: 백엔드의 `dify_client.py`는 서버 환경변수에 안전하게 숨겨진 `DIFY_API_KEY`를 HTTP 헤더에 주입하여 Dify 서버와 통신합니다. 이 로직 덕분에 브라우저 상에서는 AI 서버의 존재와 키가 완벽히 은닉됩니다.
*   **노이즈 필터링**: 추후 고도화 과정에서, LLM이 내부적으로 사고하는 과정인 `<think>` 태그나 불필요한 마크다운 기호를 서버 단에서 미리 제거하여 프론트엔드의 렌더링 부하를 줄이는 전처리 로직을 거칩니다."

### 3) 프론트엔드: 정규식을 활용한 실시간 화면 분할 파싱 로직
"마지막으로 사용자에게 보여지는 프론트엔드의 파싱(Parsing) 로직입니다.

*   **동작 방식**: AI가 모든 정보를 수집해 최종 프롬프트를 완성하면, 응답 데이터 안에 `<prompt> ... </prompt>` 라는 특수한 커스텀 태그로 결과물을 감싸서 반환하도록 설계했습니다.
*   **정규식(Regex) 파싱 로직**: 우측 결과창을 담당하는 `PromptAssembler.jsx` 컴포넌트는 전역 상태(Zustand)에 들어온 AI의 응답 메시지 중 정규식을 이용해 `<prompt>` 내부의 텍스트만 쏙 뽑아냅니다.
*   **Split View 렌더링**: 뽑아낸 프롬프트는 우측 결과창에 실시간 타이핑 효과와 함께 전문적인 형태로 렌더링되고, 태그 밖의 일상적인 챗봇 대화 내용은 좌측 대화창(`ChatInterface.jsx`)에만 남게 됩니다. 이를 통해 사용자는 과정(채팅)과 결과(프롬프트)를 명확히 구분해서 볼 수 있습니다."

## 4. 요약 및 향후 과제 (Conclusion)

**[발표 스크립트]**
"결론적으로 PromptGate는 **1) Dify의 상태 기반 워크플로우 로직으로 필수 정보를 추적하고, 2) Flask 백엔드로 통신을 캡슐화하며, 3) React 프론트엔드의 정규식 파싱으로 화면을 완벽히 분리해내는** 유기적인 시스템 구조를 갖추고 있습니다.

이러한 논리적인 아키텍처를 바탕으로, 향후에는 개인정보(PII)를 자동으로 마스킹하는 보안 로직과 토큰 절감량을 시각화하는 대시보드 로직을 추가로 결합하여 시스템의 완성도를 더욱 높일 계획입니다. 감사합니다."

## 💡 발표 대비용 예상 Q&A (Tip)

**Q. Dify의 Chatflow에서 정보가 누락된 건 어떤 기준으로 판단하나요?**
> A. "Dify 내부에 '엔터티 추출(Entity Extraction)' 노드를 설정했습니다. 사용자의 자연어 입력에서 '예산', '기간', '테마' 등의 키워드를 감지하여 내부 변수에 할당합니다. 조건 분기 노드(IF/ELSE)에서는 이 변수 배열을 순회하며 빈 값(Null)이 존재하는지 논리적으로 검사하여 역질문 분기로 빠질지 결정하게 됩니다."

**Q. 왜 AI가 프롬프트를 줄 때 `<prompt>` 태그를 쓰게 만들었나요? JSON으로 받으면 되지 않나요?**
> A. "스트리밍(Streaming) 환경을 고려한 설계입니다. AI가 응답을 생성할 때 JSON 형식으로 강제하면 데이터가 완벽히 닫힐 때까지 파싱 에러가 발생하기 쉽습니다. 반면 텍스트 기반의 커스텀 태그(`<prompt>`)를 사용하면, 프론트엔드에서 태그가 열리는 순간부터 텍스트를 실시간으로 잘라내어 끊김 없이(Typing effect) 사용자에게 보여줄 수 있어 사용자 경험(UX) 측면에서 훨씬 유리합니다."
